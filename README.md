## 📂 Featured Academic & Practice Projects

* **Hierarchical Multilayer Lightweight Intrusion Classification:** My minor project for college. The Hierarchical Multilayer Lightweight Intrusion Classification (Hi‑MLIC) project proposes a multi‑stage, lightweight machine‑learning system for detecting and classifying network intrusions in large‑scale traffic environments, implemented in Python. The goal is to accurately distinguish between benign traffic and various malicious activities while handling severe class imbalance (many more normal samples than attacks) and keeping computational cost low for real‑time deployment.

Architecture overview
Hi‑MLIC uses a three‑layer hierarchical structure:

Layer‑1: A binary classifier that separates traffic into benign vs. malicious.

Layer‑2: A coarse‑grained classifier that groups malicious traffic into four broad attack categories.

Layer‑3: A fine‑grained classifier that identifies 23 specific intrusion types, such as flood, brute‑force, and injection attacks.

This stepwise design simplifies the full multiclass problem, reduces misclassification, and focuses computation only on suspicious traffic after each layer.

Datasets and preprocessing
The model is trained on consolidated versions of the CIC‑IDS2017 and UNSW‑NB15 datasets, merged into CM‑CIC‑IDS2017 and CM‑UNSW‑NB15 to improve diversity and coverage of attack patterns. Feature‑engineering and misclassification analysis are applied to remove redundant or confusing features, which cuts model complexity and speeds up processing without hurting accuracy.

Performance and lightweight design
Hi‑MLIC achieves high detection recall (around 98.8%) and improves classification accuracy over non‑hierarchical models, especially for rare attack classes, by leveraging the hierarchical imbalance‑handling strategy. The “lightweight” aspect comes from a streamlined architecture, early filtering of benign traffic, selective feature usage, and efficient Python‑based implementation, making it suitable for resource‑constrained or real‑time network‑monitoring systems..
